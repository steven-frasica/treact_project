# Treact Replica Site Project

![Treact Project Homepage](assets/treact_project_hp.png)

In this project, I built a replica web page of this [Treact template](https://treact.owaiskhan.me/components/landingPages/SaaSProductLandingPage).

## Prerequisites

1. A modern browser (Chrome, Firefox, Safari)
2. VS Code (or another code editor)
3. Git (optional)
4. Live Server extension in VS Code (or a similar extension)

## Get Started

Using your terminal, go to the directory where you want to save the project, then run:

```bash
git clone https://github.com/steven-frasica/treact_project.git
cd treact_project
code .
```

You can enable the `code` shell command in VS Code if it is not set up yet.

## Open and View Web Page

Right-click `index.html` and select Open with Live Server.

Mac shortcut: `Cmd + L`, then `Cmd + O`  
Windows shortcut: `Alt + L`, then `Alt + O`

## Project Features

1. Multi-section responsive layout based on the Treact landing page
2. Reusable flexbox-based section structure
5. Grouped CSS selectors to reduce repetition

## Structure and Design

Using the basics of HTML and CSS learned in this course so far, I structured the web page into ten parts: nine `sections` and the `footer`. The flexbox layout method was used extensively for responsive design, making the web page accessible on various screen sizes. While building later sections of the site, patterns began to emerge, and I reused code in some instances. I also grouped CSS selectors that had the same properties in an effort to follow the DRY (Don't Repeat Yourself) principle.

## Additional Tools and Uses

I used GitHub Copilot and FES's Duck to answer questions about concepts I needed more help understanding and to build the non-required FAQs section. For example, Copilot helped build the sliding testimonials using what I had started with: one complete, static testimonial. It was also helpful in explaining and creating the `details` and `summary` elements and the built-in functionality to hide and reveal FAQ answers.

## Lessons and Next Steps

Following this project, I have a much better understanding of HTML, CSS, and Flexbox. I also became more adept at inspecting and understanding the DOM (Document Object Model) in the browser while working with the Treact template. After seeing the extensive CSS code used to slide the testimonials, I am looking forward to learning how JavaScript can implement the same feature in fewer lines of code, and how JS can further enhance a web page's capabilities.
